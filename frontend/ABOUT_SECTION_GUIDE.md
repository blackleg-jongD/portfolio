# About 섹션 가이드

## 📍 About 섹션 수정 위치

About 섹션의 내용은 다음 파일에서 수정할 수 있습니다:

```
frontend/src/data/about.json
```

## 📝 About 섹션에 넣을 좋은 내용

임베디드/로봇 개발자 포트폴리오에 적합한 내용 제안:

### 1. 자기소개 (1-2문단)
- 전공/학력
- 개발 분야
- 관심사

### 2. 주요 경험 및 강점 (2-3문단)
- 주로 다루는 기술/프로젝트
- 경력/경험
- 기술적 강점

### 3. 목표 및 비전 (1문단)
- 앞으로의 목표
- 관심 분야
- 구직 의향

## 📋 예시 템플릿

### 기본 템플릿
```json
{
  "sections": [
    {
      "type": "paragraph",
      "content": "첫 번째 문단 내용..."
    },
    {
      "type": "paragraph",
      "content": "두 번째 문단 내용..."
    },
    {
      "type": "paragraph",
      "content": "세 번째 문단 내용..."
    }
  ]
}
```

### 세부 항목이 있는 템플릿
```json
{
  "sections": [
    {
      "type": "heading",
      "content": "전공 및 관심 분야"
    },
    {
      "type": "paragraph",
      "content": "전기전자공학을 전공하며..."
    },
    {
      "type": "heading",
      "content": "주요 경험"
    },
    {
      "type": "paragraph",
      "content": "ROS2를 활용한..."
    }
  ]
}
```

## 💡 작성 팁

### 좋은 About 섹션의 특징:
1. **명확성**: 어떤 개발자인지 한눈에 알 수 있어야 함
2. **구체성**: 추상적인 표현보다는 구체적인 경험/기술 언급
3. **간결성**: 너무 길지 않게 (3-5문단 권장)
4. **차별성**: 다른 지원자와 차별되는 점 강조

### 포함하면 좋은 내용:
- ✅ 전공 및 학력 (간단히)
- ✅ 주로 다루는 기술 스택
- ✅ 주요 프로젝트 경험 (간단히)
- ✅ 기술적 강점
- ✅ 앞으로의 목표/비전
- ✅ 구직 의향 (선택)

### 피해야 할 내용:
- ❌ 너무 길고 장황한 설명
- ❌ 개인적인 사생활 (불필요한 부분)
- ❌ 부정적인 표현
- ❌ 과장된 표현

## 🔧 수정 방법

1. `frontend/src/data/about.json` 파일 열기
2. `content` 필드의 텍스트를 원하는 내용으로 수정
3. 문단을 추가하거나 삭제할 수 있음
4. 저장 후 브라우저 새로고침

## 📌 현재 설정

현재 `about.json` 파일에는 임베디드/로봇 개발자에 맞는 기본 템플릿이 들어있습니다.
원하시는 내용으로 자유롭게 수정하시면 됩니다!
